<interfaces>
The production implementation must expose this plain-data contract from `src/systems/Navigation.js`:

- `inflateAabb(aabb, radius, epsilon = 0.01)` requires a finite ordered AABB, `radius >= 0`, and `epsilon >= 0`; it returns a new `{ minX, maxX, minZ, maxZ }` and never mutates the input.
- `NavigationGrid` accepts `{ bounds, blockers, cellSize = 0.4, radius = 0.32, epsilon = 0.01 }`, requires finite strictly ordered bounds, `cellSize > 0`, `radius >= 0`, `epsilon >= 0`, and finite ordered blocker AABBs, rejects a grid larger than `250_000` cells before allocation, inflates and rasterizes static blockers once in the constructor, and exposes `worldToCell`, `cellToWorld`, `isWalkable`, `nearestWalkable`, and `findPath`.
- Grid cells use plain `{ col, row }` integer indices. `worldToCell({ x, z })` returns the containing cell or `null` outside the configured bounds; `cellToWorld(cell)` returns the cell center or `null` for an invalid cell; `isWalkable(cell)` returns `false` for invalid or occupied cells; and `nearestWalkable(point)` returns a deterministic cell or `null` when the point is outside the grid or no free cell exists.
- `NavigationGrid.findPath(start, goal)` accepts plain `{ x, z }` points and returns deterministic world-space `{ x, z }` cell centers, including the resolved start and goal cells, or `[]` when no legal route exists.
- `moveCircleAgainstAabbs(position, delta, blockers, options)` accepts plain `{ x, z }` values and `{ radius = 0.32, epsilon = 0.01, maxSubstep = 0.1 }`; it requires finite points/AABBs, `radius >= 0`, `epsilon >= 0`, and `maxSubstep > 0`, returns a new `{ x, z }`, rejects motions requiring more than `4096` substeps, resolves X then Z for sliding, and performs radius-expanded comparisons without allocating a new blocker graph each frame. If the starting center already overlaps any expanded blocker, it fails closed by returning an unchanged copy and applying no delta; Phase 2 remains responsible for selecting legal spawn/reform positions.
- Cardinal-neighbor order is fixed as north (`row - 1`), west (`col - 1`), east (`col + 1`), then south (`row + 1`). Nearest-walkable BFS uses that order. A* compares lowest `f`, then lowest heuristic, then insertion order. No diagonal edge is emitted, so a path can never cut a blocked corner.
</interfaces>
