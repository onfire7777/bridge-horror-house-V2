# Plan 02-01: Route Matt Through the House

## Goal

Make every live hunt route, door crossing, manifestation, reform, and capture obey one tested spatial model.

## Tasks

### 1. Write the failing spatial-validity contract

Add pure Node tests for segment occlusion, route-distance spawn selection, unreachable fail-closed behavior, and full-open door clearance semantics. Run the focused suite and retain the failing output before production edits.

### 2. Extend pure navigation primitives

Add deterministic route-length calculation, legal spawn selection, and segment-clearance checks to `Navigation.js`. Keep validation bounded and dependency-free.

### 3. Integrate cached waypoint pursuit and doors

Build one grid in `Game`, route `Ghost` through waypoints on a bounded cadence, request nearby doors, collide against current door state, and stop when no route exists. Keep the cardboard presentation intact.

### 4. Make manifestations and capture legal

Route stalk/finale/reform spawn selection through authored connected candidates. Evaluate capture after movement and require current line of sight so walls and closed doors block it.

### 5. Verify, review, and ship

Run focused and full tests, production build, diff checks, CBM impact/trace, and GSD review. Fix all critical and warning findings, record verification/summary evidence, then commit and push `main` with zero local/remote divergence.

## Completion Gate

- NAV-02, NAV-03, NAV-04, and NAV-05 have runnable evidence.
- Matt reaches the player by hallways/doors and never falls back to direct wall traversal.
- Closed doors remain physical until fully open.
- No legal route means no movement/capture.
